# Character Detail Page Code Review

**Goal**: Conduct thorough architectural and code quality review of character-detail feature changes against FanHub established patterns and standards.

---

## 📋 Review Framework

This review validates changes across:

- **Architecture Alignment** — Matches FanHub .NET two-tier design (ARCHITECTURE.md)
- **Backend Code Standards** — ASP.NET Core patterns, async/await, error handling, HTTP contracts
- **Frontend Code Standards** — Blazor component lifecycle, HttpClient usage, state management
- **Data Validation** — Models, DTOs, entity relationships
- **Security & Edge Cases** — Null checks, validation, error states, empty data
- **UI/UX Patterns** — Loading states, error displays, accessibility
- **Consistency** — Code style, naming, file organization

---

## 🔍 Phase 1: Architecture Alignment Check

Reference: [ARCHITECTURE.md](../dotnet/docs/ARCHITECTURE.md)

### Two-Tier Architecture Validation

**Backend (ASP.NET Core Web API)**

- [ ] Controller inherits from `ControllerBase`?
- [ ] Class-level attributes: `[ApiController]` + `[Route("api/[controller]")]` present?
- [ ] All route handlers return `Task<IActionResult>` (not `Task<T>` directly)?
- [ ] Handler names follow RESTful conventions (GetCharacter, CreateCharacter, UpdateCharacter, DeleteCharacter)?
- [ ] Dependency injection via constructor: `public controller(FanHubContext context) { _context = context; }`?

**Frontend (Blazor Server)**

- [ ] Component declared with `@page` directive if it's a routable page?
- [ ] Dependencies injected with `@inject` (HttpClient, NavigationManager)?
- [ ] Component inherits from correct pattern (not needed explicitly)?
- [ ] `@code` block exists if component has logic?
- [ ] Parameter passing uses `[Parameter]` attribute?

**Database Layer (Entity Framework Core)**

- [ ] Entity classes use proper naming (Character, Show, Episode, Quote)?
- [ ] Foreign keys follow pattern: `public int ShowId { get; set; }`?
- [ ] Navigation properties: `public virtual Show Show { get; set; }`?
- [ ] Collections use interface: `public ICollection<Episode> Episodes { get; set; }`?
- [ ] DbContext has `OnModelCreating()` for relationship configuration (if needed beyond defaults)?

## 🔍 Phase 2: Backend Code Review (ASP.NET Core)

Reference: [backend.instructions.md](../.github/instructions/backend.instructions.md)

### HTTP Status Codes & Responses

- [ ] `GetCharacter()` returns `OkObjectResult` (200) with character detail?
- [ ] `CreateCharacter()` returns `CreatedAtAction()` (201), NOT `Ok()` (200)?
- [ ] `UpdateCharacter()` returns `Ok()` (200) or `NoContent()` (204)?
- [ ] `DeleteCharacter()` returns `NoContent()` (204)?
- [ ] Missing resource (invalid ID) returns `NotFound()` (404)?
- [ ] Validation errors return `BadRequest()` (400)?
- [ ] Unhandled exceptions return `StatusCode(500, ...)` with error detail?

### Async/Await Patterns

**Critical Validation**:

```csharp
// ✅ CORRECT
var character = await _context.Characters.FirstOrDefaultAsync(c => c.Id == id);
await _context.SaveChangesAsync();
var list = await _context.Episodes.ToListAsync();

// ❌ WRONG (missing await)
var character = _context.Characters.FirstOrDefaultAsync(c => c.Id == id);
_context.SaveChangesAsync();  // Fire-and-forget bug!
```

- [ ] `FindAsync()` operations are `await`ed?
- [ ] `FirstOrDefaultAsync()`, `ToListAsync()` are `await`ed?
- [ ] `SaveChangesAsync()` is `await`ed?
- [ ] `.Include()` chains properly with async queries?
- [ ] All handler methods return `async Task<IActionResult>` (not missing async)?

### Null Safety & Error Handling

- [ ] Null check before accessing entity properties:

```csharp
var character = await _context.Characters.FindAsync(id);
if (character == null)  // ✅ Check before use
    return NotFound();
var detail = new CharacterDetailDTO { Name = character.Name, ... };  // Safe
```

- [ ] Try/catch wraps database operations?
- [ ] Invalid IDs don't throw unhandled exceptions?
- [ ] Related entities checked before property access (e.g., `character.Show.Title`)?

### DTO Mapping

- [ ] CharacterDetailDTO includes all user-facing fields: Name, Bio, Tagline, CharacterType, ActorName, Status, ImageUrl, IsMainCharacter?
- [ ] DTO includes Show, Episodes, Quotes, RelatedCharacters as nested objects?
- [ ] Null/optional fields properly handled in mapping?
- [ ] Collections (Episodes, Quotes, RelatedCharacters) initialized or null-checked?

## 🔍 Phase 3: Frontend Code Review (Blazor)

Reference: [frontend.instructions.md](../.github/instructions/frontend.instructions.md)

### Component Lifecycle & Data Loading

Verify pattern:

```razor
@page "/characters/{CharacterId:int}"
@inject HttpClient Http
@inject NavigationManager NavigationManager

@code {
    [Parameter]
    public int CharacterId { get; set; }

    private CharacterDetailDTO character;
    private bool isLoading = true;
    private string errorMessage;

    protected override async Task OnInitializedAsync()
    {
        await LoadCharacter();
    }

    private async Task LoadCharacter()
    {
        try
        {
            isLoading = true;
            errorMessage = null;
            character = await Http.GetFromJsonAsync<CharacterDetailDTO>(
                $"http://localhost:5265/api/characters/{CharacterId}");
        }
        catch (Exception ex)
        {
            errorMessage = ex.Message;
            character = null;
        }
        finally
        {
            isLoading = false;
        }
    }
}
```

- [ ] Component uses `protected override async Task OnInitializedAsync()` (not sync version)?
- [ ] Parameters decorated with `[Parameter]`?
- [ ] HttpClient injected: `@inject HttpClient Http`?
- [ ] Loading state tracked: `private bool isLoading = true`?
- [ ] Error state tracked: `private string errorMessage`?
- [ ] API call wrapped in try/catch/finally?
- [ ] Finally block sets `isLoading = false` to unblock UI?
- [ ] Character nulled on error to prevent runtime access errors?

### Conditional Rendering (Loading / Error / Success States)

- [ ] Show loading indicator while `isLoading == true`?
- [ ] Show error message if `errorMessage != null`?
- [ ] Show character detail only if `character != null`?
- [ ] No naked nulls (e.g., `@character.Name` without null guard)?

Example:

```razor
@if (isLoading)
{
    <p>Loading character details...</p>
}
else if (character != null)
{
    <h1>@character.Name</h1>
    <p>Actor: @character.ActorName</p>
    @if (!string.IsNullOrEmpty(character.Bio))
    {
        <p>@character.Bio</p>
    }
}
else if (errorMessage != null)
{
    <p style="color:red;">Error: @errorMessage</p>
}
```

- [ ] All conditional branches covered (loading, error, success)?
- [ ] Null-safe property access for optional fields (Tagline, CharacterType)?
- [ ] Empty collections (Episodes, Quotes, RelatedCharacters) handled gracefully?

### Component Composition

- [ ] Child components passed only required properties?
- [ ] Child components (CharacterHeader, EpisodeList, RelatedCharacters) have `[Parameter]` for inputs?
- [ ] Parent passes data correctly: `<CharacterHeader Character="character" />`?
- [ ] No hardcoded API URLs (relative or environment-based)?

## 🔍 Phase 4: Data Model & Validation

### Character Entity Validation

- [ ] Name: `[Required]` + `[StringLength(100)]`?
- [ ] Bio: `[StringLength(500)]`?
- [ ] Tagline: `[StringLength(200)]`?
- [ ] CharacterType: `[StringLength(50)]`?
- [ ] ActorName: `[Required]` + `[StringLength(100)]`?
- [ ] Status: `[Required]` + `[StringLength(50)]` (or enum)?
- [ ] ImageUrl: `[Url]` (if provided)?
- [ ] Show relationship: `public int ShowId { get; set; }` + `public virtual Show Show { get; set; }`?
- [ ] Episodes/Quotes/RelatedCharacters: `public ICollection<T>` initialized as `new List<T>()`?

### DTO Property Correspondence

**Verify Backend DTO matches Frontend expectations**:

CharacterDetailDTO should include:

```csharp
public class CharacterDetailDTO
{
    public int Id { get; set; }
    public string Name { get; set; }
    public string Bio { get; set; }
    public string Tagline { get; set; }
    public string CharacterType { get; set; }
    public string ActorName { get; set; }
    public string Status { get; set; }
    public string ImageUrl { get; set; }
    public bool IsMainCharacter { get; set; }
    public ShowDTO Show { get; set; }
    public List<EpisodeDTO> Episodes { get; set; }
    public List<QuoteDTO> Quotes { get; set; }
    public List<RelatedCharacterDTO> RelatedCharacters { get; set; }
}
```

- [ ] All fields match backend mapping in CharactersController.GetCharacter()?
- [ ] DTO collections initialized: `public List<T> Episodes = new()`?
- [ ] Optional fields nullable: `public string? Tagline`?

## 🔍 Phase 5: Security & Edge Cases

### Null Safety

- [ ] No direct property access without null check: `character.Show.Title` ❌ (should check Show first)
- [ ] Navigation properties guarded: `character?.Show?.Title` or explicit checks?
- [ ] Empty collections don't crash: `character.Episodes.Count()` vs `character.Episodes?.Count() ?? 0`?

### Validation Edge Cases

- [ ] Character with missing Tagline (nullable) still renders?
- [ ] Character with no Episodes shows empty list, not error?
- [ ] Character with no Quotes shows empty list, not error?
- [ ] Character with no RelatedCharacters shows empty section, not error?
- [ ] Invalid IDs (0, negative, non-existent) return 404, not 500?
- [ ] API connection timeout shows error message, not blank page?

### CORS & Configuration

- [ ] Frontend HttpClient targets correct backend URL (`http://localhost:5265` for dev)?
- [ ] URLs not hardcoded in production (use configuration)?
- [ ] CORS configured in Backend Program.cs allows Frontend origin?

## 🎯 Common Issues to Watch For

### Backend

❌ **Missing `await`**: `_context.SaveChangesAsync()` without await
❌ **No null check**: `var title = character.Show.Title` crashes if Show is null
❌ **Wrong status code**: POST returns 200 instead of 201
❌ **Fire-and-forget**: Don't ignore Task results from async methods

### Frontend

❌ **Sync lifecycle**: `OnInitialized()` instead of `OnInitializedAsync()`
❌ **Naked null access**: `@character.Name` without null guard
❌ **No error handling**: No try/catch around API calls
❌ **Blocked UI**: No `isLoading = false` in finally block

### Data Layer

❌ **Validation missing**: No `[Required]`, `[StringLength]` attributes
❌ **Relationship misconfigured**: Foreign key or navigation property missing
❌ **DTO incomplete**: Missing fields that UI needs to display
