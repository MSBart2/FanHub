## Combined Plan: Character Detail Enhancements (Tagline + Status Badges)

**TL;DR:** Add two fields to Character (`Tagline`, `CharacterType`) → single backend migration → two new badge components → shared styling. Total scope: ~150 lines of new code.

**Backend:**

- Backend/Models/Character.cs — Add `Tagline` + `CharacterType` properties
- Backend/Data/FanHubContext.cs — Seed both fields

- Backend.Tests/Models/CharacterModelTests.cs — Validation tests

**Frontend:**

## 🛠️ Implementation Order

### **Phase 1: Backend Model** (Single consolidated step)

1. **Update Character.cs** — Add both properties:

   ```csharp
   [Required]
   [StringLength(200)]
   public string Tagline { get; set; }

   public string CharacterType { get; set; }  // Permissive, multi-value format: "Hero,Antihero"
   ```

### **Phase 2: Database Migration** (Single consolidated step)

2. **Create migration** — Adds both columns in one go:
   ```bash
   dotnet ef migrations add AddCharacterTaglineAndCharacterType
   dotnet ef database update
   ```

### **Phase 3: Seed Data** (Single consolidated step)

3. **Update FanHubContext.cs** — Populate both fields for all characters:
   ```csharp
   HasData(
       new Character {
           Id = 1,
           Tagline = "Complex anti-hero with a troubled past",
           CharacterType = "Villain,Antihero"
           // ... other properties
       },
       // ... more characters
   )
   ```

### **Phase 4: Frontend Components** _(parallel with testing)_

4. **Create StatusBadge.razor** — Render Status as badge:

   ```html
   @if (!string.IsNullOrEmpty(Status)) {
   <span class="badge badge-status badge-@Status.ToLower()">@Status</span>
   } @code { [Parameter] public string Status { get; set; } }
   ```

5. **Create CharacterTypeBadge.razor** — Parse & render multiple types:

   ```html
   @if (!string.IsNullOrEmpty(CharacterType)) { @foreach (var type in types) {
   <span class="badge badge-type badge-@type.ToLower()">@type</span>
   } } @code { [Parameter] public string CharacterType { get; set; } private
   string[] types = Array.Empty<string
     >(); protected override void OnParametersSet() { types =
     string.IsNullOrEmpty(CharacterType) ? Array.Empty<string
       >() : CharacterType.Split(',').Select(t => t.Trim()).ToArray(); }
       }</string
     ></string
   >
   ```

6. **Update CharacterHeader.razor** — Integrate tagline + both badge components:

   ```html
   <div class="character-header">
     <div class="character-header-image">
       <!-- existing image code -->
     </div>
     <div class="character-header-info">
       <h1>@Character.Name</h1>

       <!-- NEW: Badges row -->
       <div class="character-badges">
         <StatusBadge Status="@Character.Status" />
         <CharacterTypeBadge CharacterType="@Character.CharacterType" />
       </div>

       <!-- NEW: Tagline -->
       @if (!string.IsNullOrEmpty(Character.Tagline)) {
       <p class="character-tagline">@Character.Tagline</p>
       }

       <div class="character-meta">
         <!-- existing meta -->
       </div>
     </div>
   </div>
   ```

### **Phase 5: Styling** (Single consolidated step)

7. **Add badge CSS** — Covers both Status & CharacterType badges:

   ```css
   .character-badges {
     margin: 0.5em 0 0.8em 0;

   }
   ```

### **Phase 6: Testing** _(parallel with Phases 4-5)_

8. **Add model validation tests** — Test Tagline constraints:
   - Tagline required (not null)
   - Tagline max 200 chars enforced
   - Empty string rejected

9. **Add integration test** — Verify API response:
   - GET /api/characters/{id} returns both Tagline and CharacterType

10. **Manual verification**:
    - Run app → navigate to character detail page
    - Verify tagline appears under name
    - Verify Status badge displays
    - Verify CharacterType badges display (multiple if present)
    - Test null-safe rendering (no crash if fields missing)
